- **Popular-routes card: dead decorative arrow → valid + accessible** — each card in `PopularRoutes` (`src/main.tsx`) is a single `<a href="/flights?from=…&to=…">`, but the circular arrow overlaid on the image was a `<button aria-label="Open {from} route">` with **no `onClick`** nested **inside** that anchor. Three real defects: (1) invalid HTML — `<button>` is interactive content and is forbidden as a descendant of `<a>`; (2) a **dead keyboard tab stop** — keyboard users tab onto the arrow and pressing Enter/Space does nothing (the anchor navigates only on itself), exactly the owner's "buttons that don't work" family; (3) the `aria-label` competed with the anchor's own accessible name for screen-reader users even though the arrow is purely decorative. Fix: converted the `<button>` to `<span className="route-go" aria-hidden="true">` (decorative, non-focusable, out of the a11y tree) and renamed the three `.route-card button` CSS selectors → `.route-card .route-go` (base + transition + `:hover` scale). Because the generic `.route-card span` rule would otherwise leak `margin-top:3px; color:#61748a` onto the new span, added `margin:0; color:#07162c` (house ink) plus `pointer-events:none` to the base rule so the arrow stays pixel-identical (same 44px white circle, same position/shadow/hover-scale) and clicks pass straight through to the card link. Verified: `npm run build` green (`tsc --noEmit` 0 errors + `vite build` 0 errors); compiled `dist` CSS confirms the `.route-card .route-go` rule emitted correctly and **zero** `.route-card button` left in the bundle; DeepSeek + MiMo both reviewed and approved (no remaining issue; runs saved in `docs/agent-runs/`).
